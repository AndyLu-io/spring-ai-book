第 11 章

多模态 Agent

理解多模态 Agent 的架构设计，掌握图像、语音、视频等多种模态的处理方法，使用 Spring AI 构建支持多模态输入的 Agent 系统

预计 2-3 周 7 个知识点 高级难度

📚 开篇故事：全能助手的五感

想象你有一个全能助手。你递给他一张菜单，他能看懂上面的菜名和价格；你对他说一句话，他能听懂你的需求；你给他一段视频，他能理解里面发生了什么。他不仅会"听"和"看"，还能画图、朗读、识别人脸。

这就是多模态 Agent。传统的 AI Agent 只能处理文字 -- 你打字输入，它打字回复。但多模态 Agent 拥有了"五感"：它能看图（视觉理解）、听话（语音识别）、说话（语音合成）、读文档（OCR）、生成图片（图像生成）。就像人类用眼睛看、耳朵听、嘴巴说来全方位感知世界一样，多模态 Agent 用不同的"感官"来理解和服务用户。

在本章中，我们将学习如何为 Agent 装上这些"感官"，让它从一个"只会打字的书呆子"进化为一个能看、能听、能说、能画的多才多艺助手。我们将用 Spring AI 实现图像理解、语音交互和多模态工具调用。

🎯 学习目标

在本章结束后，你将能够：

  • 理解多模态架构 — 掌握多模态 Agent 的整体架构设计，理解模态编码器、融合层的作用
  • 处理图像输入 — 能够使用 Spring AI 处理图像理解、OCR、视觉问答等任务
  • 处理语音输入 — 掌握语音识别（ASR）和语音合成（TTS）的集成方法
  • 构建多模态 Agent — 能设计和实现支持多种输入模态的 Agent 应用

📄 第1节：多模态基础 -- AI 的五感

📄 什么是多模态？

模态（Modality）是指信息的表达方式。人类通过多种模态感知世界：视觉（看）、听觉（听）、触觉（触）、嗅觉（闻）、味觉（尝）。AI 也正在获得类似的"多感官"能力：

  • 文本模态 -- 最基础的模态，LLM 的原生能力。理解输入文字、生成输出文字。
  • 视觉模态（Vision） -- 理解图像内容。包括图像分类、目标检测、场景理解、OCR 等。
  • 听觉模态（Audio） -- 语音识别（ASR）将语音转文字，语音合成（TTS）将文字转语音。
  • 视频模态（Video） -- 理解视频内容，结合视觉和时间序列分析。需要从视频中抽帧再逐帧理解。

📄 视觉理解（Vision）

视觉理解是多模态 Agent 最核心的"感官"。现代 LLM（如 GPT-4V、Claude 3、Gemini）都具备了视觉能力。

图像输入处理流程：

  • 图像预处理 -- 缩放、裁剪、格式转换。大多数模型接受 JPEG/PNG，建议分辨率控制在 1024x1024 以内。
  • 图像编码 -- 视觉编码器（如 ViT）将图像转换为 Token 序列。一张 224x224 的图像会被分割为 196 个 patch，每个 patch 变成一个 Token。
  • 视觉-语言对齐 -- 通过对比学习（如 CLIP）将图像特征和文本特征映射到同一语义空间，使 LLM 能"看懂"图像。
  • 推理与生成 -- LLM 基于图像 Token 和文本 Token 的融合表示进行推理，生成文字描述或回答问题。
✔ 关键概念：图像 Token 化

图像不是直接被 LLM "看"的，而是被编码器转换成一串数字向量（Token），就像把照片"翻译"成 LLM 能理解的"语言"。一张图片可能占用 85-1000 个 Token，具体取决于分辨率。

📄 语音识别与合成（ASR/TTS）

ASR（Automatic Speech Recognition）将用户语音转为文字：

  • OpenAI Whisper -- 开源、支持多语言，精度高，适合离线场景。支持 50+ 种语言。
  • 阿里 Paraformer -- 实时流式识别，中文表现优异，适合国内部署场景。
  • Google Cloud Speech-to-Text -- 云端服务，支持实时流式和批量两种模式。

TTS（Text-to-Speech）将 Agent 的文字回复转为语音：

  • OpenAI TTS -- 支持 6 种声音，语速可调，自然度高。
  • Edge TTS -- 免费、低延迟，支持多种中文音色。
  • CosyVoice -- 阿里开源，支持声音克隆，可自定义音色。

📄 视频理解基础

视频理解是视觉理解的延伸。目前主流方案是抽帧 + 视觉模型：

视频输入  →  按间隔抽帧  →  逐帧视觉理解  →  时序融合  →  综合描述
  │              │                  │                │
  ▼              ▼                  ▼                ▼
30fps视频    每2秒取1帧          GPT-4V分析每帧     LLM综合时间线
100帧        50张图片             每帧的内容        生成完整描述

视频理解的关键挑战：(1) 帧数爆炸 -- 1分钟视频有1800帧，不可能全部发送给模型；(2) 时序关系 -- 需要理解事件的先后顺序；(3) 成本控制 -- 每帧都是一个图像 Token，大量帧会导致 Token 超限。

📄 多模态融合策略

将不同模态的信息有效融合，有三种主要策略：

策略方式适用场景代表模型
早期融合在输入层合并不同模态特征模态间交互密切GPT-4V、Gemini
晚期融合各模态独立编码后拼接模态相对独立CLIP、BLIP
混合融合多层级融合，兼顾全局和局部复杂多模态任务Flamingo、GPT-4o

在 Spring AI 应用中，我们主要使用晚期融合策略 -- 将图像、音频分别通过各自的编码器处理后，以 Token 形式拼接到 LLM 的输入中。这是目前最成熟、最容易工程化的方式。

✔ 要点小结

多模态让 AI Agent 从"只会打字"进化为"能看能听能说"。视觉理解是核心能力，语音是重要的交互模态，视频理解是视觉的延伸。在 Spring AI 中，我们主要通过多模态 LLM（如 GPT-4V）和专用工具（Whisper、TTS）来实现多模态能力。

📄 视觉推理链（Visual Chain-of-Thought）

普通视觉模型只给"一个答案"。Visual CoT 让模型先描述看到什么，再推理结论，就像医生看X光片："先描述影像特征，再给出诊断"。

普通模式：
输入: [图片] + "这张图里有什么？"
输出: "一只猫和一只狗在沙发上"

Visual CoT 模式：
输入: [图片] + "请先描述你看到的，然后分析场景"
输出:
  描述阶段: "我看到一张沙发，沙发上有一只橘色猫和一只黑色狗。
              猫在左边蜷缩着，狗在右边趴着。背景是客厅。"
  分析阶段: "这是一个温馨的家居场景。猫和狗相处融洽，
              说明它们是同一家的宠物。可能是午后的休息时间。"

Visual CoT 的价值：可解释性更强（你能看到模型的推理过程）、准确率更高（中间步骤减少了"跳跃"导致的错误）、易于调试（出了问题能定位到具体步骤）。

📄 截图分析 Agent

一个典型的实战场景：Agent 接收用户截图，自动分析界面内容并给出改进建议。

// ============================================ // 截图分析 Agent - 分析 UI 截图并给出改进建议 // ============================================ @Service public class ScreenshotAnalyzerAgent { private final ChatClient chatClient; public ScreenshotAnalyzerAgent(ChatClient chatClient) { this.chatClient = chatClient; } /** * 分析 UI 截图，给出设计建议 * * @param imageUrl 截图 URL * @param question 用户的具体问题 */ public String analyze(String imageUrl, String question) { return chatClient.prompt() .system(""" 你是一位资深 UI/UX 设计师。 当用户给你截图时，请： 1. 先描述你看到的界面元素（Visual CoT） 2. 从以下维度分析： - 布局合理性 - 色彩搭配 - 字体和间距 - 用户体验 - 可访问性（无障碍） 3. 给出具体的改进建议（带优先级） 输出格式： ## 界面描述 [描述内容] ## 问题分析 [按维度列出问题] ## 改进建议 [按优先级排列建议] """) .user(u -> u .text(question) .media(Media.fromUri(imageUrl))) // 发送图片 .call() .content(); } }

📄 OCR 与文档理解

OCR（Optical Character Recognition）是将图像中的文字转换为可编辑文字的技术。结合 LLM 的文档理解能力，可以实现智能文档处理。

// ============================================ // 文档理解 Agent - 从图片中提取和理解文字 // 适用场景：发票识别、合同分析、证件读取 // ============================================ @Service public class DocumentUnderstandingAgent { private final ChatClient chatClient; /** * 从图片中提取结构化信息 * 示例：上传发票图片 → 返回结构化的发票数据 */ public InvoiceData extractInvoice(String imageUrl) { String result = chatClient.prompt() .system(""" 你是一个发票识别专家。 从用户提供的发票图片中提取以下信息，返回 JSON： { "invoiceNumber": "发票号码", "date": "开票日期", "seller": "销售方名称", "buyer": "购买方名称", "items": [ {"name": "商品名", "amount": 金额, "tax": 税额} ], "totalAmount": "合计金额", "totalTax": "合计税额" } 如果某个字段无法识别，填 null。 """) .user(u -> u .text("请识别这张发票") .media(Media.fromUri(imageUrl))) .call() .content(); return objectMapper.readValue(result, InvoiceData.class); } /** * 合同条款分析 * 上传合同图片 → AI 自动分析关键条款和风险点 */ public ContractAnalysis analyzeContract(String imageUrl) { String result = chatClient.prompt() .system(""" 你是一个法律合同分析师。 从图片中读取合同条款，分析并返回： { "parties": ["甲方", "乙方"], "effectiveDate": "生效日期", "expiryDate": "到期日期", "keyTerms": ["关键条款1", "关键条款2"], "risks": ["风险点1", "风险点2"], "summary": "合同摘要" } """) .user(u -> u .text("请分析这份合同") .media(Media.fromUri(imageUrl))) .call() .content(); return objectMapper.readValue(result, ContractAnalysis.class); } }
⚠️ 常见陷阱

陷阱 1：图片过大导致 Token 爆炸。 一张高分辨率图片可能消耗上千 Token。发送前务必压缩到合理尺寸（建议宽度不超过 1024px），或使用模型支持的自动缩放功能。

陷阱 2：过度依赖视觉模型的 OCR。 视觉模型的 OCR 能力不如专用 OCR 引擎（如 Tesseract）。对于精确的文字提取需求，建议先用 OCR 引擎处理，再将文字发送给 LLM 理解。

陷阱 3：忽略图像质量的影响。 模糊、倾斜、低对比度的图像会严重影响识别效果。在发送前可以做预处理：灰度化、二值化、倾斜校正。

✔ 要点小结

Vision-Based Agent 让 Agent 具备了"看"的能力。核心要点：(1) 图像通过编码器转换为 Token 序列发送给多模态 LLM；(2) Visual CoT 提高可解释性和准确率；(3) OCR + LLM 实现智能文档处理；(4) 注意图像大小和质量对效果的影响。

// ============================================ // 图像生成工具 - 注册为 Agent 的 @Tool // ============================================ @Service public class ImageGenerationTools { private final OpenAiImageClient imageClient; @Tool(description = "根据文字描述生成图片。用于创作、设计、可视化等场景。") public ImageResult generateImage( @ToolParam(description = "图片的文字描述，越详细效果越好") String prompt, @ToolParam(description = "图片尺寸：1024x1024, 1024x1792, 1792x1024") String size) { ImageGenerationResponse response = imageClient.call( new OpenAiImageOptions() .model("dall-e-3") .size(size) .quality("hd") .prompt(prompt)); return new ImageResult( response.getResult().getOutput().getUrl(), prompt ); } @Tool(description = "编辑现有图片：根据原图和文字描述生成新图。") public ImageResult editImage( @ToolParam(description = "原图 URL") String imageUrl, @ToolParam(description = "编辑描述") String editPrompt) { // 下载原图 → 调用编辑 API → 返回新图 byte[] originalImage = downloadImage(imageUrl); ImageGenerationResponse response = imageClient.edit( originalImage, editPrompt); return new ImageResult( response.getResult().getOutput().getUrl(), editPrompt); } }

📄 语音交互工具

语音工具让 Agent 能"听"和"说"，实现语音对话能力。

// ============================================ // 语音交互工具 - ASR + TTS // ============================================ @Service public class VoiceTools { private final SpeechClient speechClient; // OpenAI TTS private final WhisperClient whisperClient; // OpenAI Whisper ASR @Tool(description = "将文字转为语音。用户要求语音回复时使用。") public AudioResult textToSpeech( @ToolParam(description = "要转换的文字") String text, @ToolParam(description = "声音类型：alloy, echo, fable, onyx, nova, shimmer") String voice) { byte[] audio = speechClient.synthesize(text, voice, "mp3"); String audioUrl = audioStorageService.upload(audio); return new AudioResult(audioUrl, Duration.ofSeconds(text.length() / 10)); } @Tool(description = "将语音转为文字。用户发送语音消息时使用。") public String speechToText( @ToolParam(description = "音频文件 URL") String audioUrl, @ToolParam(description = "音频语言，如 zh, en") String language) { byte[] audio = downloadAudio(audioUrl); return whisperClient.transcribe(audio, language); } }

📄 多模态 RAG（Visual RAG）

传统 RAG 只检索文本。多模态 RAG 可以同时检索图文混合内容，例如根据文字描述找到相关图片，或根据图片找到相关文字文档。

图文混合检索

  • 查询：文字 → 匹配图文
  • 查询：图片 → 匹配相关文档
  • Embedding：CLIP 统一编码
  • 存储：向量数据库 + 元数据

视觉文档理解

  • PPT/PDF：提取页面图片 + 文字
  • 表格：识别表格结构并转为数据
  • 图表：理解折线图、柱状图
  • 公式：识别并解释数学公式
// ============================================ // 多模态 RAG - 使用 CLIP 统一编码图文 // ============================================ @Service public class MultimodalRagService { private final ClipEmbeddingClient clipClient; private final VectorStore vectorStore; /** * 索引图文混合文档 * 用 CLIP 将图片和文字映射到同一向量空间 */ public void indexDocument(String text, String imageUrl) { // CLIP 编码文字和图片到同一向量空间 float[] textEmbedding = clipClient.encodeText(text); float[] imageEmbedding = clipClient.encodeImage(imageUrl); // 融合两个向量（加权平均或拼接） float[] fusedEmbedding = fuseEmbeddings( textEmbedding, imageEmbedding, 0.5f); // 存储到向量数据库 Document doc = new Document(text, Map.of( "imageUrl", imageUrl, "type", "multimodal" )); doc.setEmbedding(fusedEmbedding); vectorStore.accept(List.of(doc)); } /** * 多模态检索 * 支持文字查图文、图片查图文 */ public List<Document> search(String query) { float[] queryEmbedding = clipClient.encodeText(query); return vectorStore.similaritySearch( SearchRequest.query(query) .withTopK(5) .withSimilarityThreshold(0.7f)); } }
⚠️ 常见陷阱

陷阱 1：图像生成的 Prompt 工程。 DALL-E 和 Stable Diffusion 的 Prompt 与 LLM Prompt 不同，需要使用英文、描述性语言。用 "A serene mountain lake at sunset, oil painting style" 而不是 "画一个日落湖景"。

陷阱 2：多模态 RAG 的 Embedding 不一致。 用不同模型编码的文字和图片无法在向量空间中匹配。必须使用 CLIP 等联合编码模型来统一编码。

陷阱 3：语音工具的延迟问题。 ASR + LLM + TTS 的链式调用延迟很高。考虑使用流式 ASR 和流式 TTS 来减少用户等待时间。

✔ 要点小结

多模态工具让 Agent 能生成图片、分析图像、处理语音。多模态 RAG 用 CLIP 实现图文统一检索。设计要点：(1) 图像生成需要专门的 Prompt 工程；(2) 多模态工具可以组合使用，Agent 自动选择模态；(3) 多模态 RAG 的关键是图文统一编码。

💻 Spring AI 实战：多模态 Agent 完整实现

📄 多模态消息构建（核心 API）

Spring AI 通过 Media 类来构建多模态消息。这是所有多模态交互的基础。

// ============================================ // Spring AI 多模态消息构建 - 三种图像输入方式 // ============================================ @Service public class MultimodalChatService { private final ChatClient chatClient; /** * 方式1: 通过 URL 发送图片（最常用） * 适用于：网络图片、云端存储的图片 */ public String analyzeByUrl(String imageUrl, String question) { return chatClient.prompt() .user(u -> u .text(question) .media(Media.fromUri(imageUrl))) // 关键：.media() 添加图片 .call() .content(); } /** * 方式2: 通过本地文件发送图片 * 适用于：本地测试、文件上传场景 */ public String analyzeByFile(Resource imageResource, String question) { return chatClient.prompt() .user(u -> u .text(question) .media(Media.fromResource(imageResource))) // 从 Resource 加载 .call() .content(); } /** * 方式3: 通过 Base64 发送图片 * 适用于：前端直接上传的图片数据 */ public String analyzeByBase64(String base64Data, String mimeType, String question) { return chatClient.prompt() .user(u -> u .text(question) .media(Media.fromBase64(mimeType, base64Data))) // Base64 编码 .call() .content(); } /** * 多图对比：一次发送多张图片 * 例如：对比两张产品图、分析多张报表 */ public String compareImages(String url1, String url2, String question) { return chatClient.prompt() .user(u -> u .text(question) .media(Media.fromUri(url1)) // 图片1 .media(Media.fromUri(url2))) // 图片2 .call() .content(); } }

📄 完整多模态 Agent

将所有多模态工具整合到一个 Agent 中，实现"能看能听能说能画"的全能助手。

// ============================================ // 完整的多模态 Agent // 整合图像理解、图像生成、语音交互、OCR // ============================================ @Service public class MultimodalAgent { private final ChatClient chatClient; private final ImageGenerationTools imageGenTools; private final ImageAnalysisTools imageAnalysisTools; private final VoiceTools voiceTools; public MultimodalAgent( ChatClient chatClient, ImageGenerationTools imageGenTools, ImageAnalysisTools imageAnalysisTools, VoiceTools voiceTools) { this.chatClient = chatClient; this.imageGenTools = imageGenTools; this.imageAnalysisTools = imageAnalysisTools; this.voiceTools = voiceTools; } /** * 处理多模态输入 * 用户可以发送文字、图片、语音的任意组合 */ public String process(MultimodalInput input) { return chatClient.prompt() .system(""" 你是一个多模态 AI 助手，具备以下能力： 1. 看图理解 - 分析图片内容、识别文字、检测物体 2. 图像生成 - 根据描述创建图片 3. 语音交互 - 将文字转为语音、将语音转为文字 4. 文档理解 - 从图片中提取结构化信息 当用户发送图片时，先用视觉能力分析图片内容， 再结合文字问题给出回答。如果需要生成图片或语音， 自动调用相应工具。 """) .user(u -> { var builder = u.text(input.text()); for (String imageUrl : input.imageUrls()) { builder = builder.media(Media.fromUri(imageUrl)); } return builder; }) .tools(imageGenTools, imageAnalysisTools, voiceTools) .call() .content(); } } // 多模态输入记录 public record MultimodalInput( String text, List<String> imageUrls ) { public static MultimodalInput of(String text) { return new MultimodalInput(text, List.of()); } public static MultimodalInput of(String text, String imageUrl) { return new MultimodalInput(text, List.of(imageUrl)); } }

🎓 面试高频题

👨‍💻 Q1: 多模态 LLM 和传统 NLP 模型在处理图像时有什么本质区别？

传统 NLP 模型只能处理文字，处理图像需要先用 OCR 提取文字再输入给模型，丢失了视觉信息（颜色、布局、空间关系）。多模态 LLM（如 GPT-4V）通过视觉编码器（如 ViT）将图像编码为 Token 序列，与文本 Token 在同一语义空间中联合推理。这意味着模型能"看到"图像的视觉特征，而不仅仅是"读"文字。Spring AI 通过 Media 类封装了这种图像输入，开发者无需关心底层编码细节。

👨‍💻 Q2: 如何设计一个支持图像理解的 RAG 系统？

核心挑战是图文统一编码。方案：(1) 使用 CLIP 等联合编码模型，将文字和图片映射到同一向量空间；(2) 索引阶段，对每张图片提取 OCR 文本 + CLIP 图像向量，融合后存入向量数据库；(3) 检索阶段，用户查询用 CLIP 编码后做相似度搜索；(4) 生成阶段，将检索到的图文对发送给多模态 LLM 生成回答。关键是选择合适的融合策略（加权平均或拼接）和相似度阈值。

👨‍💻 Q3: 多模态 Agent 的性能瓶颈在哪里？如何优化？

主要瓶颈：(1) 图像 Token 化 -- 一张图占 85-1000 Token，发送多张图会快速消耗上下文窗口；(2) 图像生成延迟 -- DALL-E 生成一张图需要 5-15 秒；(3) 语音链路延迟 -- ASR + LLM + TTS 串行调用。优化策略：(1) 图像压缩和智能裁剪，只发送关键区域；(2) 图像生成使用异步任务 + 进度回调；(3) 语音使用流式 ASR/TTS，边生成边播放；(4) 缓存高频请求的视觉分析结果。

👨‍💻 Q4: Spring AI 中如何构建多模态消息？

Spring AI 通过 Media 类支持多模态消息构建。在 ChatClient 的 .user() 方法中，使用 Lambda 构建器：.user(u -> u.text("问题").media(Media.fromUri(imageUrl)))。支持三种图像来源：Media.fromUri()（URL）、Media.fromResource()（本地文件）、Media.fromBase64()（Base64 编码）。可以多次调用 .media() 一次发送多张图片。底层会自动将图像编码为模型所需的 Token 格式。

👨‍💻 Q5: 视频理解目前有哪些主流方案？各自的优缺点？

主流方案：(1) 抽帧 + 视觉模型 -- 每隔 N 秒抽帧，逐帧用 GPT-4V 分析。优点：实现简单、可复用现有视觉模型；缺点：帧数多时成本高、丢失时序信息。(2) 专用视频模型（如 Gemini 1.5 Pro）-- 原生支持视频输入。优点：理解时序关系、效率更高；缺点：支持的视频时长有限。(3) 视频 + 语音联合分析 -- 抽帧 + ASR 分别处理视觉和音频，最后融合。适合会议记录、教学视频等场景。工程上最常用的是方案1。
