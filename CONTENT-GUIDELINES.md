# 学习手册内容增强标准

## 核心原则

### 1. 详细度要求
- 每个知识点必须包含：**概念解释 → 原理图示 → 代码示例 → 实战应用 → 常见陷阱**
- 代码示例必须完整可运行，包含必要的 import 和配置
- 每个代码块后必须有详细注释和解释

### 2. 趣味性设计
- **比喻法**：用生活中的例子解释技术概念
  - LLM → 压缩了全人类知识的超级大脑
  - RAG → 带着课本考试的学生
  - Agent → 能独立思考的机器人助手
  - Prompt → 和 AI 沟通的魔法咒语
  - Token → AI 的语言货币单位
  - 向量数据库 → 给 AI 配备的超级图书馆

- **案例故事**：用真实场景引入
  - 每章开头用一个"如果...会怎样"的故事
  - 用对话形式展示 AI 的能力

- **小贴士框**：
  - 绿色：实用技巧 "小窍门"
  - 黄色：常见坑 "踩坑警告"
  - 蓝色：延伸知识 "你知道吗"
  - 紫色：面试考点 "面试高频"

### 3. 可学性优化
- **渐进式学习路径**：每个章节分 3-4 个小节，由浅入深
- **动手实验**：每小节后有"试试看"练习
- **思维导图**：每章开头有知识图谱
- **对比表格**：技术选型时用表格对比
- **速查手册**：常用 API 和配置速查表

### 4. 典型性筛选
- 只包含最常用 20% 的知识点（覆盖 80% 场景）
- 优先选择 Spring AI 官方推荐的方式
- 避免过度学术化，聚焦工程实践

## 章节结构模板

```
1. 章节开篇故事/场景引入（200字）
2. 学习目标清单（5-8个）
3. 知识图谱（思维导图）
4. 小节1：核心概念
   - 概念解释（比喻+定义）
   - 原理图示
   - 代码示例（完整可运行）
   - 试试看（练习）
5. 小节2-N：类似结构
6. 章节小结（要点回顾）
7. 常见面试题
8. 下一章预告
```

## 视觉设计

- 使用 emoji 增加趣味性（章节标题、列表项）
- 代码块使用深色主题，语法高亮
- 重要信息用卡片样式突出
- 流程图使用 ASCII art（兼容性好）
